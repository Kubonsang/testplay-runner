# TestPlay Runner v0.6.0-beta — Release Notes

**Release date:** 2026-04-03
**Previous release:** v0.5.1-beta

---

## Summary

v0.6.0-beta is the network-ready release. It adds the infrastructure needed for multi-instance network test scenarios: per-instance environment variable injection, Windows process tree kill, artifact retention, failure excerpts for faster AI triage, and phase accuracy improvements.

**Impact on agent-driven workflows:** agents can now declare per-instance env vars in the scenario file (no CLI flag juggling), immediately see the assertion message + source location in `excerpt` without parsing logs, and rely on honest phase reporting (`compiling -> done`) in single-phase mode.

---

## What's New

### 1. Failure excerpt (`internal/parser`)

Failed test entries now include an `excerpt` field combining the assertion message with the source location for immediate triage.

**Format:**
```
"Expected 2 but was 3 (at MyTests.cs:42)"
```

When no file info is available in the stack trace, the excerpt falls back to the raw message:
```
"something broke"
```

- Present only on failed tests (`omitempty`)
- Absent for passing and skipped tests
- Uses `filepath.Base()` for the filename (no full path)

### 2. Phase accuracy (`internal/unity/executor`)

Single-phase mode no longer emits a misleading `running` phase after Unity exits.

**Before (v0.5.1):**
```
compiling -> running -> done
```

**After (v0.6.0):**
```
compiling -> done
```

Two-phase mode is unchanged (`compiling -> running -> done`), where `running` accurately marks the transition from compile to test phase.

### 3. Env injection for scenario instances (`internal/scenario`, `internal/unity`)

Scenario JSON instances now support an `env` map of environment variables injected into the Unity process.

```json
{
  "instances": [
    {
      "role": "host",
      "config": "host/testplay.json",
      "env": { "NETWORK_ROLE": "server", "PORT": "7777" }
    },
    {
      "role": "client",
      "config": "client/testplay.json",
      "depends_on": "host",
      "env": { "NETWORK_ROLE": "client", "SERVER_PORT": "7777" }
    }
  ]
}
```

**Injection flow:** Scenario JSON -> `InstanceSpec.Env` -> `ProcessRunner.Env` -> `MergeEnv(os.Environ(), env)` -> `exec.Cmd.Env`

**Validation:**
- Empty env keys are rejected at `Load()` time
- Keys containing `=` are rejected (would produce malformed env entries)

**Platform-aware key matching:** `MergeEnv` uses case-sensitive key comparison on Unix and case-insensitive (`strings.EqualFold`) on Windows, matching each OS's env semantics. Implemented via build-tagged `envkey_unix.go` / `envkey_windows.go`.

### 4. Windows process tree kill (`internal/unity/runner_windows`)

Windows `setSysProcAttr` is no longer a no-op. It now:
- Creates the Unity process in a new process group (`CREATE_NEW_PROCESS_GROUP`)
- Overrides `cmd.Cancel` to run `taskkill /F /T /PID <pid>` on context cancellation
- Sets `WaitDelay = 5s` for cleanup after kill

If `taskkill` fails (most commonly because the process already exited), it logs to stderr and returns `os.ErrProcessDone` to avoid redundant cleanup.

### 5. Artifact retention (`internal/config`, `internal/history`, `internal/artifacts`)

New `retention.max_runs` config controls automatic cleanup of old run results and artifact directories.

```json
{
  "retention": {
    "max_runs": 30
  }
}
```

| Setting | Behavior |
|---------|----------|
| Omitted / `null` | Default 30 |
| `0` | Pruning disabled |
| `50` | Keep newest 50 |
| `-1` | Rejected (config error) |

**Prune behavior:**
- Runs after each single-mode `testplay run`, after JSON output is written
- Best-effort: prune failures are silently ignored (never affects exit code)
- Only removes entries matching the run-ID format (`YYYYMMDD-HHMMSS-xxxxxxxx`); non-run-ID files/directories are preserved
- Defensive guard: `Prune(keep <= 0)` returns immediately (no mass deletion)
- Scenario mode does not auto-prune (deferred to v0.7+)

### 6. Shared run-ID validation (`internal/runid`)

New `internal/runid` package extracts the run-ID format regex, previously duplicated in `history` and `artifacts` packages.

## Output Schema Changes

| Field | Location | Change |
|-------|----------|--------|
| `excerpt` | `tests[].excerpt` | New (string, optional) — present only for failed tests |
| `retention.max_runs` | `testplay.json` | New config field (int pointer, optional, default 30) |

---

## CLI Changes

No new flags or commands. Scenario JSON gains the `env` field (additive, non-breaking). Config gains `retention` block (additive, non-breaking).

---

## What Is NOT Supported in This Release

- **Exit 6 (build failure)** and **exit 7 (permission)** — documented but not yet returned
- **Scenario-mode artifact retention** — each instance has its own stores; pruning complexity deferred to v0.7+
- **Streaming log write failure detection** — `stdout.log`/`stderr.log` write errors during Unity execution are not caught
