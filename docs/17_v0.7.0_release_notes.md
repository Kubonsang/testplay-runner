# TestPlay Runner v0.7.0 — Release Notes

**Release date:** 2026-04-03
**Previous release:** v0.6.0-beta

---

## Summary

v0.7.0 is the release candidate milestone. It lowers the barrier to entry with `testplay init` for config generation, adds GoReleaser-powered pre-built binaries for all major platforms, introduces cross-platform CI, and completes scenario-mode artifact retention that was deferred from v0.6.0.

**Impact on agent-driven workflows:** agents can now bootstrap a project with `testplay init` (no manual JSON authoring), download pre-built binaries instead of compiling from source, and rely on automatic artifact cleanup in both single-mode and scenario-mode runs.

---

## What's New

### 1. `testplay init` command (`cmd/testplay/init.go`)

New CLI command that generates a `testplay.json` configuration file with sensible defaults.

```bash
testplay init --unity-path /path/to/Unity
testplay init --test-platform play_mode
testplay init --force  # overwrite existing config
```

**Unity path resolution:** `--unity-path` flag > `UNITY_PATH` env var > empty (with warning in both stderr and stdout `warnings` field).

**Validation:**
- Rejects invalid `--test-platform` values (exit 5) — only `"edit_mode"` and `"play_mode"` accepted
- Refuses to overwrite existing `testplay.json` unless `--force` is specified (exit 5)
- Uses the `--config` flag path for output location (default: `testplay.json` in cwd)

**Generated config includes:** `schema_version`, `unity_path`, `project_path`, `test_platform`, `timeout.total_ms` (300000), `result_dir`, `retention.max_runs` (30).

**Testability:** Uses `initDeps` struct pattern (consistent with `runDeps`/`scenarioDeps`) for dependency injection in tests.

### 2. GoReleaser — pre-built binaries (`.goreleaser.yml`, `.github/workflows/release.yml`)

Cross-platform binary distribution triggered by `v*` tag push.

**Platforms:**

| OS | Arch | Archive |
|----|------|---------|
| darwin | amd64, arm64 | tar.gz |
| linux | amd64, arm64 | tar.gz |
| windows | amd64 | zip |

`windows/arm64` is excluded (unsupported by Unity).

**Build injection:** `ldflags` inject `version`, `commit`, and `date` into the binary at build time with correct `v` prefix (`-X main.version=v{{.Version}}`).

**Release settings:** `prerelease: auto` (tags with `-rc`, `-beta` suffixes are marked as pre-releases).

### 3. Cross-platform CI (`.github/workflows/test.yml`)

`go test ./...` and `go build ./cmd/testplay` on every push to `main` and every pull request.

**Matrix:** ubuntu-latest, macos-latest, windows-latest.
**Go version:** auto-detected from `go.mod` via `go-version-file`.
**Optimization:** `push` trigger restricted to `main` branch (with `tags-ignore: v*`) to avoid duplicate workflow runs when a PR branch is pushed.

### 4. Scenario-mode artifact retention (`cmd/testplay/run.go`)

Artifact pruning now runs after scenario-mode (`--scenario`) runs, completing the retention feature introduced in v0.6.0 for single-mode only.

**Deduplication:** when multiple instances share the same project (same `result_dir` + artifact root), pruning runs only once per unique project. Implemented via a `seen` map with `resultDir|artifactRoot` composite key.

**Lazy-load fallback:** when configs are not available (test-injected runner), configs are loaded on demand with `Validate(false)` (skipping `unity_path` check) — best-effort, load failures silently skipped.

### 5. Race condition fix — dependency ready/done channel (`internal/scenario/runner.go`)

Fixed a race condition where Go's `select` could randomly pick `depDoneCh` over `depReadyCh` when both were closed simultaneously (host signaled ready, then its goroutine exited). The `depDoneCh` case now re-checks `depReadyCh` before declaring fast-fail.

## Output Schema Changes

| Field | Location | Change |
|-------|----------|--------|
| `created` | `testplay init` stdout | New — path to generated config file |
| `warnings` | `testplay init` stdout | New (array, optional) — present when `unity_path` is empty |
| `retention.max_runs` | generated `testplay.json` | Now included in init output (default 30) |

---

## CLI Changes

| Change | Details |
|--------|---------|
| New command | `testplay init [--unity-path] [--test-platform] [--force]` |
| Command count | 5 → 6 (`init` added) |
| New exit scenarios | `testplay init` returns exit 5 for existing file or invalid platform |

---

## What Is NOT Supported in This Release

- **Exit 6 (build failure)** and **exit 7 (permission)** — documented but not yet returned
- **Network test configuration** — NGO/Mirror harness integration (deferred to v1.0+)
