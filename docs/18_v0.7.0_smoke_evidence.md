### 4. Race condition fix — dependency ready/done channel (`internal/scenario/runner.go`)

| Test | File | Verifies |
|---|---|---|
| `TestRunScenario_ClientStartsAfterHostReady` | `runner_test.go` | Client starts after host signals ready; 10/10 runs pass with `-race` |

**Root cause:** Go's `select` picks randomly when multiple cases are ready. When host closed `readyCh` then exited (closing `doneCh`), the client's select could pick `doneCh` and fast-fail with exit 4.

**Fix:** In the `<-depDoneCh` case, re-check `depReadyCh` with a nested non-blocking select before declaring fast-fail.

**Before (broken):**
```go
case <-depDoneCh:
    // immediately fast-fail
```

**After (fixed):**
```go
case <-depDoneCh:
    select {
    case <-depReadyCh:
        // was actually ready — proceed
    default:
        // truly not ready — fast-fail
    }
```

  and config inspection. No Windows runtime tests exist.
