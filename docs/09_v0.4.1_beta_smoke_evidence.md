## Notes

- All orchestration verification is unit-test-based with injected fake runners.
  Real Unity execution with `--scenario` requires two Unity projects with
  separate `testplay.json` configs — this is not exercised in the automated
  test suite but has been manually validated against the fixture project.
- The race detector (`go test -race`) passes cleanly across all 10 packages,
  confirming no data races in the channel-based synchronization logic.
