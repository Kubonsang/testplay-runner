# TestPlay Runner v0.8.0 — Release Notes

**Release date:** 2026-04-23
**Previous release:** v0.7.1

---

## Summary

v0.8.0 — "The Honest Contract" — closes the gap between the documented agent
contract and actual runtime behavior. Three weaknesses called out in v0.7.x
README under "Known Limitations" are fixed:

1. `testplay list` is no longer guaranteed-incomplete; it can now return the
   full inventory from a run cache and signals completeness explicitly.
2. Exit codes 6 and 7, previously documented but never emitted, are now
   returned for the conditions they describe.
3. `testplay-status.json` carries a monotonic `seq` field so polling agents
   can detect whether the file changed without parsing timestamps.

**Impact on agent-driven workflows:** agents can now branch on `complete` to
decide whether to trust `list` output, distinguish license/build-target
failures from C# compile failures (exit 6 vs exit 2), distinguish shadow
permission errors from generic dependency errors (exit 7 vs exit 1), and
poll status with a single integer comparison instead of timestamp parsing.

No backward-incompatible changes — all new fields are additive.

## What's New

### 1. `seq` field in `testplay-status.json` (`internal/status/status.go`)

`status.Writer` now maintains an internal counter that increments on every
`Write()` call. The counter is injected into the `Status.Seq` field before
the file is written.

```json
{
  "schema_version": "1",
  "seq": 7,
  "phase": "running",
  "run_id": "20260423-120000-a3f8b2c1",
  "updated_at": "2026-04-23T12:00:05Z"
}
```

**Polling pattern:**

```python
last_seq = 0
while True:
    s = json.load(open("testplay-status.json"))
    if s["seq"] != last_seq:
        last_seq = s["seq"]
        # process update
    time.sleep(0.1)
```

The previous polling pattern (parsing `updated_at`) still works; `seq` is
purely additive. Each `status.Writer` instance has its own independent
counter, so scenario-mode `testplay-status-<role>.json` files have
per-instance sequences.

### 2. Exit 6 — Unity license / build-target failure (`internal/unity/compiler.go`, `internal/unity/executor.go`)

Unity's CLI emits no NUnit XML and no C# compile errors when activation
fails or when a required build module isn't installed. Pre-v0.8.0, this
silently degraded to exit 2 ("compile failure"), causing agents to attempt
source fixes for an environment problem.

`ParseBuildFailure()` matches Unity's stderr against a small set of
case-insensitive patterns:

```go
var buildFailurePatterns = []*regexp.Regexp{
    regexp.MustCompile(`(?i)no valid unity license`),
    regexp.MustCompile(`(?i)failed to acquire.*license`),
    regexp.MustCompile(`(?i)license:.*no valid`),
    regexp.MustCompile(`(?i)is not (installed|supported)`),
    regexp.MustCompile(`(?i)module.*\bmissing\b`),
}
```

The check is wired in two places in `executor.go`:

- **Single-phase / parseResults path** — when no XML is produced and no
  compile errors are extracted, fall through to `ParseBuildFailure` before
  defaulting to exit 2.
- **Two-phase / compile-phase path** — when the compile-only Unity
  invocation exits non-zero with no parseable errors, check stderr for
  build failure signals.

**JSON output (exit 6):**

```json
{
  "schema_version": "1",
  "exit_code": 6,
  "tests": [],
  "errors": []
}
```

Agents should branch on exit 6 to surface a license/module activation
message, not a source-edit prompt.

### 3. Exit 7 — shadow workspace permission error (`internal/runsvc/service.go`)

Shadow workspace preparation walks the project directory, creates the
`.testplay-shadow-<run_id>/` tree, and copies `Assets/`/`ProjectSettings/`.
Filesystem permission failures during this phase previously surfaced as
exit 1 (generic dependency error) or exit 9 (runner system error),
giving agents no signal to fix permissions specifically.

```go
ws, wsErr = shadow.Prepare(ctx, req.Config.ProjectPath, runID, opts)
if wsErr != nil {
    if errors.Is(wsErr, os.ErrPermission) {
        return Response{ExitCode: 7}, fmt.Errorf("runsvc: prepare shadow workspace: %w", wsErr)
    }
    return Response{}, fmt.Errorf("runsvc: prepare shadow workspace: %w", wsErr)
}
```

The error chain through `shadow.Prepare → workspace.copyDir →
workspace.copyFile` preserves `os.ErrPermission` via `fmt.Errorf("...%w...",
err)` wrapping, so `errors.Is` works across the boundary.

`cmd/testplay/run.go` was updated to forward `resp.ExitCode` even when
infrastructure errors are present, so exit 7 reaches the shell instead of
being clamped to 1.

### 4. `testplay list` run cache (`internal/listcache/`, `cmd/testplay/list.go`)

After a successful `testplay run` (exit 0 or 3), the full test inventory
parsed from NUnit XML is written atomically to
`<project>/.testplay/cache/list.json`:

```json
{
  "schema_version": "1",
  "cached_run_id": "20260423-120000-a3f8b2c1",
  "cached_at": "2026-04-23T12:01:00Z",
  "tests": ["MyTests.PlayerTests.TestJump", "MyTests.PlayerTests.TestRun"]
}
```

`testplay list` now tries the cache first:

```json
{
  "schema_version": "1",
  "complete": true,
  "source": "run_cache",
  "cached_run_id": "20260423-120000-a3f8b2c1",
  "tests": ["MyTests.PlayerTests.TestJump", "MyTests.PlayerTests.TestRun"]
}
```

When no cache exists, the prior C# static scan runs and is marked
explicitly incomplete:

```json
{
  "schema_version": "1",
  "complete": false,
  "source": "static_scan",
  "tests": ["MyTests.PlayerTests.TestJump"]
}
```

Agents can branch on `complete` to decide whether to trust the list for
filter generation. Custom test attributes (`[NetworkTest]` etc.) that the
static scanner cannot detect are present in the run cache because Unity
itself enumerated them during execution.

**Scenario-mode safety:** list cache write-back is skipped when
`SkipCacheWriteBack` is set, matching the Library cache discipline — each
scenario instance runs a different subset of tests, so writing back from
any single instance would corrupt the cache for the others.

## Schema Changes

| File | Field | Type | Notes |
|---|---|---|---|
| `testplay-status.json` | `seq` | int | Monotonically increasing per writer; starts at 1 |
| `testplay list` output | `complete` | bool | `true` when from run cache, `false` when from static scan |
| `testplay list` output | `source` | string | `"run_cache"` or `"static_scan"` |
| `testplay list` output | `cached_run_id` | string | Present only when `source == "run_cache"` |
| `.testplay/cache/list.json` | (new file) | object | Run cache produced after exit 0/3 |

All additions are additive; agents that ignore the new fields continue to
work unchanged.

## Upgrade Notes

No migration steps required. New fields are additive; existing agents
continue to work unchanged. To take advantage of v0.8.0:

- **Switch polling to `seq`** — replace `updated_at` parsing with integer
  comparison.
- **Branch on exit 6** — surface "fix Unity license / install build module"
  to the user instead of "fix source code".
- **Branch on exit 7** — surface "fix project directory permissions" to the
  user instead of "fix Unity environment".
- **Trust `complete: true` from `testplay list`** — when the field is
  present and true, the list is the full Unity-discovered inventory; when
  false, the list is a best-effort C# static scan.
