### 5. Artifact retention

#### 5A. Config validation (`internal/config`)

| Test | File | Verifies |
|---|---|---|
| `TestValidate_RetentionDefaults` | `config_test.go` | Omitted -> `*MaxRuns = 30` |
| `TestValidate_RetentionExplicit` | `config_test.go` | `max_runs: 50` -> `*MaxRuns = 50` |
| `TestValidate_RetentionZero_DisablesPruning` | `config_test.go` | `max_runs: 0` -> `*MaxRuns = 0` (pruning disabled) |
| `TestValidate_RetentionNegative_Rejected` | `config_test.go` | `max_runs: -1` -> `ErrConfigInvalid` |

**`*int` pointer semantics:**

| JSON value | After `Validate()` | Pruning |
|-----------|-------------------|---------|
| omitted | `*MaxRuns = 30` | Enabled (default) |
| `0` | `*MaxRuns = 0` | Disabled |
| `50` | `*MaxRuns = 50` | Enabled |
| `-1` | error | N/A |

## Notes

- All verification is unit-test-based with injected fake runners.
- Windows process kill is verified via cross-compilation (`GOOS=windows GOARCH=amd64`)
  and code inspection. No Windows runtime tests exist.
- The `internal/runid` package has no test files — it exports a single compiled
  regex and a one-line `IsValid()` function, both exercised indirectly by
  `history` and `artifacts` tests.
- The `*int` pointer approach for `MaxRuns` adds a `nil` state that JSON
  unmarshal handles correctly: omitted field -> nil -> `Validate()` defaults to 30.
